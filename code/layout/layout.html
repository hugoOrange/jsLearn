文档流布局(stream)

这是最基本的布局方式，就是按照文档的顺序一个一个显示出来，块元素独占一行，行内元素共享一行，这个相信大家都比较熟悉了，就不再赘述了

浮动/流体布局(liquid)

使用float实现的简单布局：通过float属性把元素与文档流隔离，再对主内容使用margin进行偏移。

定位布局(position)

我们也可以通过 position 属性来进行定位。

常用的属性包括 absolute relative 和 fixed ，主要是用left等位移属性进行定位。

弹性盒子布局(flex box)

弹性布局是指通过调整其内元素的宽高，从而在任何显示设备上实现对可用显示空间最佳填充的能力。弹性容器扩展其内元素来填充可用空间，或将其收缩来避免溢出。

弹性盒子是css3的新属性。

flex 的使用方法很简单，只需要将其 display 属性设置为 flex 就可以，也可以设置行内的 flex，记得 Webkit 内核的浏览器，必须加上 -webkit 前缀。注意，设为 Flex 布局以后，子元素的 float、clear 和 vertical-align 属性将失效。

.ele{
    display: -webkit-flex;
    display: flex;
    display: inline-flex;
    display: -webkit-inline-flex;
}

基本概念

父容器有以下属性：

  • - flex-direction 确立主轴方向
  • - flex-wrap 子元素溢出时
  • - flex-flow flex-direction 属性和 flex-wrap 属性的简写形式
  • - justify-content 一行的弹性项目沿主轴如何排布
  • - align-items 每行的弹性项目沿交叉轴如何排布
  • - align-content 多行的弹性项目沿主轴如何排布

align-items 和 align-content 的区别在于前者相当于对每个容器内元素进行对齐，而后者是对一行进行处理

子容器也有 6 个属性：

  • - order 子容器的排列顺序
  • - flex-grow 子容器所占空间的比例
  • - flex-shrink 子容器超出空间的压缩比例（父容器设置flex-wrap为nowrap才能生效）
  • - flex-basis 子容器在不伸缩情况下的原始尺寸
  • - flex 子元素的 flex 属性是 flex-grow,flex-shrink 和 flex-basis 的简写
  • - align-self 定义了单个弹性项目在交叉轴上应当如何对齐，这个定义会覆盖由 align-items 所确立的默认值

弹性盒子的优势在于：

  • - 页面能够适应不同的屏幕尺寸和不同的显示设备时，元素可预测的运行

弹性盒子的劣势在于：

  • - 兼容性，chrome21+, opera12.1+, IE10+, Safari6.1+, Firefox22+
